Packngo initial prompt-  Build Brief for *Pack\&Go* (Frontend Monorepo)
*One‑liner / Motto:* “Pack\&Go — you travel impromptu, cuz we plan.”
*Goal:* Generate a production‑ready, responsive *web + mobile* frontend for a modern travel super‑app with AI trip planning, a social marketplace (Travel Circles + curated trips), local experiences/workations, smart expense splits, and a gamified travel layer.

## Output Requirements

  * apps/web: *Next.js * (App Router, TypeScript, shadcn/ui)
* *Shared packages*:

  * packages/ui: cross‑platform components (Tailwind + react-native-web compatibility)
  * packages/config: tsconfig/eslint/prettier/tailwind config
  * packages/types: shared DTOs & zod schemas
  * packages/sdk: typed API client (OpenAPI or hand‑rolled) with React Query hooks
* *CI‑ready*: ESLint, Prettier, typecheck, unit tests (Vitest/Jest), basic Playwright e2e for web
* *Docs*: README.md + ARCHITECTURE.md + comments in code showing where to integrate backend APIs




---




## Tech & Libraries




* *UI*: Tailwind CSS, shadcn/ui (web), react-native + tamagui/nativewind (mobile), Framer Motion (web), Reanimated (mobile)
* *State/Data*: TanStack Query, Zustand for UI state, Zod for validation
* *Routing*: Next.js App Router (web), Expo Router (mobile)
* *Auth*: Email OTP (passwordless) with pluggable adapter (Clerk/Supabase/Auth.js) — put clear TODOs for backend URLs
* *Realtime*: WebSockets/SSE client stubs (Ably/Pusher/Socket.IO) for chat, polls, notifications
* *PWA* (web): installable, offline cache (Workbox), prefetching, responsive images
* *i18n*: next-intl on web, i18next on mobile (scaffold only)
* *A11y*: aria labels, keyboard nav, color contrast checks
* *Analytics*: event emitter with providers (PostHog/GA4) – central track() util with typed events




---




## Design System & UX




* *Look/feel*: lively, modern, card‑first; generous white space; soft shadows; rounded‑2xl; subtle gradients; glassy overlays; micro‑interactions
* *Themes*: light/dark, brand colors (primary teal, accent coral, neutral slate)
* *Motion*:




  * Page transitions (slide/fade)
  * Button/CTA hover springs
  * Story viewer progress bars
* *Responsive*: mobile‑first; breakpoints for tablet and desktop; bottom tabbar on mobile, left nav on desktop

## Core Screens (Web & Mobile parity where sensible)




1. *Onboarding & Auth*




   * Phone/email OTP; capture home city, budget band, interests (treks, cafes, culture, etc.)
2. *Home / Discover*




   * Card grid of: AI Planner CTA, curated trips (brands/influencers), Travel Circles near you, experiences/events, workations
3. *AI Planner (Conversational)*




   * Chat‑like UI: user prompt (e.g., “5 days Bali, ₹50k, adventure + cafes”), assistant replies with day‑by‑day itinerary
   * Sidebar: constraints (budget, dates, visa, weather, flights/hotels availability) + live adjustments
   * Buttons: “Refine”, “Shuffle day”, “Optimize budget”, “Book pieces”
4. *Marketplace*




   * Filters: date range, budget, theme (trek/festival/food), organizer type
   * Listing detail: itinerary snippet, inclusions, map, host profile, join/book CTA
5. *Travel Circles*




   * Create/join peer‑listed journeys (like BlaBlaCar for leisure)
   * Circle detail: members, itinerary, chat, polls, expense group link
6. *Experiences & Events*




   * Aggregation (BookMyShow, Ticketmaster, Meetup) – cards with date/time, location, price, tags
7. *Workations*




   * Listings with Wi‑Fi speed, desk photos, community vibe, cost, availability
8. *Trip Detail*




   * Multi‑day plan with drag‑reorder, map of POIs, weather badge, attachments (tickets/receipts)
   * “Add to Circle”, “Invite friends”, “Export to calendar”
9. *Expenses*




   * Group ledger, SMS/receipt auto‑parse stubs, split modes (equal/weighted/rotating), UPI/PayTM/GPay settlement buttons
10. *Stories / Social Proof*




* IG‑style story components: trip highlights, people met, places pinned; share/export




11. *Travel Graph & Gamification*




* Dashboard with yearly recap: places, spend, companions; badges (“Trekker”, “Culture Vulture”, “Eco Explorer”), leaderboards




12. *Notifications*




* Realtime updates (trip changes, poll results, expense reminders)




13. *Profile & Settings*




* Preferences, saved places, payment methods, privacy, notification toggles
